# MVP Scope

This document defines the scope of the initial release of Jirita.

The MVP focuses on delivering the core functionality required to replace the current project management workflow while maintaining a simple, modern, and highly usable experience.

Features not listed in this document should be considered out of scope unless explicitly approved.

---

# MVP Goals

The MVP should allow teams to:

- Organize work by project
- Plan and track milestones
- Create and manage tickets
- Assign work to team members
- Track time spent on work
- Store project documentation
- Generate operational reports

The MVP should be usable by real teams on real projects from day one.

---

# Projects

Projects are the primary organizational unit within Jirita.

## Capabilities

- Create project
- Edit project
- Archive project
- Assign team members
- View project activity
- View project progress

## MVP Notes

Projects should be simple to create and manage.

No advanced configuration should be required.

---

# Milestones

Milestones represent phases, deliveries, releases, or significant objectives within a project.

## Capabilities

- Create milestone
- Edit milestone
- Archive milestone
- Associate tickets with milestones
- Track milestone progress
- View milestone activity

## MVP Notes

Milestones should provide clear visibility into project progress without introducing unnecessary complexity.

---

# Tickets

Tickets are the primary unit of work within Jirita.

## Capabilities

- Create ticket
- Edit ticket
- Archive ticket
- Assign ticket
- Change status
- Add comments
- Add labels
- Associate with milestones
- Associate with projects

## MVP Notes

Ticket creation and management should be optimized for speed and simplicity.

---

# Team Management

Manage users participating in projects.

## Capabilities

- Invite users
- Remove users
- Assign users to projects
- Assign users to tickets
- View user activity

## MVP Notes

The MVP focuses on simple team participation and assignment.

Advanced permissions and role management are out of scope.

---

# Notes & Wiki

Store project-related knowledge and documentation.

## Capabilities

- Create notes
- Edit notes
- Organize notes by project
- Search notes
- Store operational documentation
- Store credentials and references

## MVP Notes

The objective is to provide lightweight project documentation without requiring a separate knowledge management platform.

---

# Time Tracking

Track time spent on work.

## Capabilities

- Log time against tickets
- Edit time entries
- View personal time entries
- View project time entries
- View milestone time entries

## MVP Notes

Time tracking should be fast and frictionless.

Recording time should never become a burden for users.

---

# Reporting

Provide operational visibility into projects and team activity.

## Capabilities

- Hours by project
- Hours by milestone
- Hours by user
- Hours by date range
- Open tickets
- Closed tickets
- Project activity

## MVP Notes

Reports should prioritize clarity and usability over complexity.

---

# Search

Search is a first-class feature in Jirita.

## Capabilities

- Search projects
- Search milestones
- Search tickets
- Search notes
- Search users

## MVP Notes

Users should be able to find information quickly without navigating multiple screens.

---

# Out of Scope

The following features are intentionally excluded from the MVP:

- Client portal
- GitHub integration
- AI-powered features
- Notifications
- Capacity planning
- Skills matrix
- Staffing management
- Scrum-specific workflows
- Story points
- Sprint management
- Repository hosting
- Internal messaging system

These features may be considered in future releases but should not impact MVP decisions.

---

# MVP Success Criteria

The MVP is considered successful when:

- Teams can manage real projects using Jirita.
- Teams can replace their existing workflow.
- Common actions require fewer clicks than competing tools.
- Users can become productive with minimal training.
- The platform feels fast, modern, and intuitive.

The MVP should prioritize usability and adoption over feature count.
